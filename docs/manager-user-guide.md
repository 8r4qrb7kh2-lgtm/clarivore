# Clarivore Manager User Guide

This guide replaces every earlier Clarivore manager guide.

Every screenshot in this guide shows one action only. The orange arrow text tells the manager exactly what to click, type, or review in that step.

All screenshots were captured from the `demo-menu` restaurant with real manager permissions. The guide starts from a fresh restaurant foundation, then continues on the same restaurant after it has been fully built out and used by a larger set of demo diners with different diets, allergens, and dish behavior. That later activity is what makes the dashboard statistics, request queue, and tablet screens look realistic.

## How To Use This Guide

1. Follow each workflow from top to bottom.
2. Do only the one action shown in each screenshot before moving to the next step.
3. Use the editor workflows first if you are setting up or updating restaurant data.
4. Use the dashboard and tablet workflows for day-to-day operations after the menu has been built and published.

## What Managers Should Remember

1. The webpage editor is where restaurant data is built and maintained.
2. The viewer is how diners see that data before they send a notice.
3. The dashboard is where managers handle messages, requests, confirmation, brand review, and analytics.
4. The server and kitchen tablets are operational notice queues, not menu editing tools.
5. The same restaurant data entered in the editor drives the later dashboard, analytics, and tablet workflows.
