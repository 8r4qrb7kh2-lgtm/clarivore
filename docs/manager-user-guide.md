# Clarivore Manager User Guide

This guide replaces every earlier Clarivore manager guide.

Every screenshot in this guide shows one action only. The orange arrow tells the manager exactly what to click, type, or review in that step.

All screenshots were captured from the `demo-menu` restaurant with real manager permissions. The guide starts with a new restaurant foundation, then uses the same restaurant after realistic manager, diner, dashboard, and tablet activity was seeded into the demo.

## How To Use This Guide

1. Follow each workflow from top to bottom.
2. Do the one action shown in the screenshot before moving to the next step.
3. If your live restaurant has more menu pages or brand items than the demo, repeat the same step on each additional card.
4. Use the editor workflows first, then use the dashboard and tablet workflows for day-to-day operations.

## What Managers Should Remember

1. The webpage editor is where restaurant data is built, corrected, confirmed, and published.
2. Recipe text becomes ingredient rows, and ingredient rows drive the diner-facing dish details.
3. Brand items can be reassigned from saved items, added as new scans, replaced from the dashboard, or appealed when no package exists.
4. Monthly confirmation has two parts: menu-page verification first, then brand-item verification.
5. The viewer, dashboard, server tablet, and kitchen tablet all depend on the restaurant data entered in the editor.
